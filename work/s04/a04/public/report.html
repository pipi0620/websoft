S09

1.From my own experience, one security issue is insecure deserialization. Google Chrome now has provide users with strong encryption password when creating a new account. It can prevent user from external attackers.
Another problem is insufficient logging and monitoring. When we have a strong monitoring function, we can quickly view the web traffic statistics and see if an exception has occurred in the system.
Also, there is cross-site scripting xss. Sometimes when we enter certain webpages, the browser will remind us that the webpage may be at risk. This is because a malicious attacker inserts malicious html code into a web page. When a user browses the page, the html code embedded in the web will be executed, thereby achieving the special purpose of the malicious user.
2.

  • Never store unfiltered user input in the database.
  • Do not create SQL statements by concatenating strings involving user input.
  • Use Secure Sockets Layer to save sensitive information.
  • Do not store any critical information in cookies.Never store unfiltered user input in the database.
